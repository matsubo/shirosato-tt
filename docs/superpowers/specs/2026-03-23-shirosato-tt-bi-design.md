# しろさとTT200 第11回 BI分析ダッシュボード 設計書

## 概要

第11回しろさとTT200（2026年3月22日開催）のレースリザルト・ラップタイムデータを分析するBIダッシュボード型Webサービス。

- **スコープ**: 2026年第11回大会の1大会分のみ
- **参考**: @kokikr氏のインフォグラフィック（ダークテーマダッシュボード）
- **品質基準**: AI TRI+ (ai-triathlon-result.teraren.com) に恥じないクオリティ

## データソース

- **リザルトPDF**: `https://shirosato-tt.com/result/result_shiroatott202603.pdf`
  - 3カテゴリ: 200km(35周), 100km(18周), 50km(9周)
  - フィールド: 総合順位, No., 氏名, 年齢, 性別, 都道府県, 総合記録, P分(ペナルティ), 男子順位, 女子順位, 年齢区分, 区分順位, 平均速度
  - ステータス: OPEN(オープン参加), DNF(リタイア), DNS(未出走)
  - 注: OPEN参加者は男子順位・女子順位・区分順位がnullになる
  - 注: ペナルティ(P分)は総合記録に既に加算済み
- **ラップタイムPDF**: `https://shirosato-tt.com/result/result_shiroatott202603_laptime.pdf`
  - フィールド: No., 氏名, 周回数, 記録, P分, 最終タイム, LAP1〜LAP35

## ページ構成

### 1. トップページ = マクロ分析ダッシュボード (`/`)

大会全体の統計を一覧表示するBIダッシュボード。

#### KPIカード (上部横並び)
- 参加者数 (エントリー総数)
- 完走者数
- 完走率 (%)
- 最速タイム (200km/100km/50km それぞれ)
- 平均タイム
- DNF数

#### カテゴリ別タイム分布 (ヒストグラム)
- 200km / 100km / 50km それぞれの完走タイム分布
- @kokikrインフォグラフィックのような色分け棒グラフ (シアン/グリーン/オレンジ)

#### TOP10テーブル + TOP3ラップ推移
- 各カテゴリのTOP10を表形式で表示 (順位, No., 氏名, タイム, 平均速度, 偏差値)
- TOP3のラップタイム推移を折れ線グラフで重ね表示

#### 統計分析セクション
- **年齢区分別参加者分布**: 棒グラフ (10代〜60代)
- **都道府県別参加者マップ or 棒グラフ**: 地域分布
- **性別比率**: ドーナツチャート
- **完走タイム偏差値分布**: ベルカーブ + ヒストグラム
- **前半 vs 後半ペース散布図**: X=前半平均ラップ, Y=後半平均ラップ, 対角線基準でネガティブ/ポジティブスプリット可視化
- **ラップ安定性ランキング**: 変動係数(CV)が低い選手TOP10

#### 選手検索・一覧
- No. / 氏名 / カテゴリで検索・フィルタ
- クリックで選手詳細ページへ遷移

### 2. 選手詳細ページ = ミクロ分析 (`/athletes/[no]`)

個別選手の詳細分析ページ。`generateStaticParams` で全選手分を事前生成。

#### 選手プロフィールカード
- No., 氏名, 年齢, 性別, 都道府県
- カテゴリ, 総合順位, カテゴリ内順位, 年齢区分順位
- 総合記録, 平均速度

#### ラップタイム推移グラフ
- 折れ線グラフ: 各ラップのタイム推移
- 平均ラインとベストラップのハイライト
- 移動平均の重ね表示 (200km: 5周, 100km: 3周, 50km: 3周)

#### 偏差値レーダーチャート
- 5軸: 総合タイム偏差値, ラップ安定性偏差値, 後半維持率偏差値, ベストラップ偏差値, 平均速度偏差値
- カテゴリ内での相対評価
- 偏差値計算式: `hensachi = 50 - 10 * (x - mean) / stddev` (タイムは低いほど良いため符号反転)

#### ペーシング分析
- 前半/後半の平均ラップ比較バー
- ネガティブ/ポジティブスプリット判定
- 後半落ち込み率 (%)
- ベストラップ / ワーストラップとその周回番号

#### ラップ安定性
- 標準偏差
- 変動係数 (CV)
- 各ラップの平均からの乖離率バーチャート (色分け: 平均より速い=グリーン, 遅い=レッド)

#### パーセンタイル表示
- カテゴリ内で上位何%か
- 年齢区分内で上位何%か

#### CdA推定ツール
- 入力フォーム: 平均パワー(W), 体重(kg)
- 環境条件は固定値表示 (2026年3月22日城里の気象データ、出典: 気象庁)
- コース条件: 1周 5.6663km (198.32km / 35周), フラット
- 推定パラメータ: Crr=0.004, ドライブトレイン効率=0.97
- 出力: 推定CdA (m^2), ラップ別CdA推移グラフ
- URLパラメータで共有可能 (`?power=200&weight=70`)
- 注意書き: 風の影響により推定値には誤差が含まれる旨を表示

#### AIコメント
- 選手のデータに基づく分析コメント (事前生成、ソースコード内JSON)
- レース戦略の評価、強み/弱みの指摘、改善提案

#### DNF/DNS選手の表示
- **DNF**: プロフィールカード + 完走した周回分のラップ推移のみ表示。偏差値・パーセンタイル・CdAは非表示とし「リタイア」バッジを表示
- **DNS**: プロフィールカードのみ表示。「未出走」バッジを表示

## PDFパーススクリプト

`scripts/parse-pdf.ts` として実装:
1. リザルトPDFをダウンロード → テーブルデータ抽出 → results.json生成
2. ラップタイムPDFをダウンロード → テーブルデータ抽出 → laptimes.json生成
3. 開発時に手動実行 (`npx tsx scripts/parse-pdf.ts`)
4. パース後にバリデーション:
   - results.json と laptimes.json の No. フィールドが一致するか検証
   - ラップタイム合計 ≈ 総合記録の検証 (ペナルティ考慮)
   - DNF選手のラップ数 < カテゴリ規定周回数の検証
5. パース精度が不十分な場合のフォールバック: 手動CSV入力 → JSON変換

## CdA計算ロジック

クライアントサイドで以下の計算を実行:

```
速度 v = コース距離(5.6663km) / ラップタイム(s)
P_effective = ユーザー入力パワー(W) * 0.97  (ドライブトレイン効率)
P_roll = Crr * m * g * v  (Crr=0.004, g=9.81, m=体重kg)
P_grade = 0 (フラットコース)
P_aero = P_effective - P_roll
CdA = 2 * P_aero / (rho * v^3)  (rho=1.2407 kg/m^3)
```

**既知の制限事項** (UIに注記として表示):
- 各ラップ内での速度変化 (コーナーでの減速・加速) は考慮していない
- 風の影響 (向かい風/追い風) によりラップ間のCdA推定値にばらつきが生じる
- 入力パワーはラップ平均値を仮定 (パワーメーターの実測平均パワーの入力を推奨)
